OVERVIEW: AI coding assistant

Examples:
  # Interactive mode
  pi

  # Interactive mode with initial prompt
  pi "List all .ts files in src/"

  # Include files in initial message
  pi @prompt.md @image.png "What color is the sky?"

  # Non-interactive mode (process and exit)
  pi -p "List all .ts files in src/"

  # Continue previous session
  pi --continue "What did we discuss?"

  # Use different model
  pi --provider openai --model gpt-4o-mini "Help me refactor this code"

  # Limit model cycling to specific models
  pi --models claude-sonnet,claude-haiku,gpt-4o

  # Limit to a specific provider with glob pattern
  pi --models "github-copilot/*"

  # Cycle models with fixed thinking levels
  pi --models sonnet:high,haiku:low

  # Start with a specific thinking level
  pi --thinking high "Solve this complex problem"

  # Read-only mode (no file modifications possible)
  pi --tools read,grep,find,ls -p "Review the code in src/"

  # Export a session file to HTML
  pi --export ~/.pi/agent/sessions/--path--/session.jsonl
  pi --export session.jsonl output.html

Environment Variables:
  ANTHROPIC_API_KEY       - Anthropic Claude API key
  ANTHROPIC_OAUTH_TOKEN   - Anthropic OAuth token (alternative to API key)
  OPENAI_API_KEY          - OpenAI GPT API key
  GEMINI_API_KEY          - Google Gemini API key
  GROQ_API_KEY            - Groq API key
  CEREBRAS_API_KEY        - Cerebras API key
  XAI_API_KEY             - xAI Grok API key
  OPENROUTER_API_KEY      - OpenRouter API key
  ZAI_API_KEY             - ZAI API key
  PI_CODING_AGENT_DIR - Session storage directory (default: ~/.pi/agent)

Available Tools (default: read, bash, edit, write):
  read, bash, edit, write, grep, find, ls

USAGE: pi-coding-agent [<options>] [<raw-messages> ...]

ARGUMENTS:
  <raw-messages>          Messages and @file paths

OPTIONS:
  --provider <provider>   Provider name
  --model <model>         Model ID
  --api-key <api-key>     API key (defaults to env vars)
  --system-prompt <system-prompt>
                          System prompt
  --append-system-prompt <append-system-prompt>
                          Append text or file contents to the system prompt
  --mode <mode>           Output mode: text (default), json, or rpc
  -c, --continue          Continue previous session
  -r, --resume            Select a session to resume
  --thinking <thinking>   Thinking level: off, minimal, low, medium, high, xhigh
  --no-session            Don't save session (ephemeral)
  --session <session>     Use specific session file
  --session-dir <session-dir>
                          Directory for session storage and lookup
  -m, --models <models>   Comma-separated model patterns for Ctrl+P cycling
  --tools <tools>         Comma-separated list of tools to enable
  --hook <hook>           Load a hook file (can be used multiple times)
  --tool <tool>           Load a custom tool file (can be used multiple times)
  -p, --print             Non-interactive mode: process prompt and exit
  --export <export>       Export session file to HTML and exit
  --no-skills             Disable skills discovery and loading
  --skills <skills>       Comma-separated glob patterns to filter skills
  --list-models           List available models (with optional fuzzy search)
  --version               Show the version.
  -h, --help              Show help information.
